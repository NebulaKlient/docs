<Accordion title="Usage Notes" icon="sparkles">
  <Expandable title="Commands">
    - **/n routes help**: prints a help message
    - **/n routes load**: reloads current routes from file
    - **/n routes edit**: toggles edit mode that allowing you to place nodes
    - **/n routes edit placewarp**: places Etherwarp node

      <Expandable title="Content">
        - /n routes edit placewarp [additional delay] [waitforsecret] [manual mode]
        - <u>Example</u>: /n routes edit placewarp 0 true false
      </Expandable>
    - **/n routes edit clear [radius]**: clears <u>EVERY</u> node in the selected radius, /n routes edit clear in the selected radius clears <u>EVERY</u> node in the room
  </Expandable>
  <Expandable title="Nodes list">
    - **Etherwarp:** places an Etherwarp node that automatically etherwarps you to the selected block using an algorithm or hardcoded yaw & pitch values
    - **Walk**: places a Walk node that automatically walks in the selected direction
    - **Pearl**: places a Pearl node that automatically throws ender pearl¬†
    - **Rotate**: places a Rotate node that automatically rotates u
    - **Chat**: places a Chat node that automatically sends message or command in chat

      <Expandable title="Explained">
        - **vclip**: /n dungeons vclip -\*
        - **pearlclip**: /n dungeons vclip -\*

        To switch from VClip to PearlClip, you need to enable VClip Auto Pearl in DungeonUtils module
      </Expandable>
    - **Press Key**: places a Press Key node that allows you to use ingame keybinds for the routes
    - **Use Item**: places a Use Item node that automatically uses selected item

    <Expandable title="Explained">
      To set rotation for <u>Pearl </u>& <u>Use Item</u> nodes, you need to <u>place a Rotate node</u> that executes first with ‚ÄúWait for Rotation‚Äù
